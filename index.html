Git (произносится «гит») — распределённая система управления версиями или просто система контроля версий. Зачем это нужно? Git позволяет следить и управлять изменениями версий вашей программы/кода в течение времени и даёт возможность вернуться к любой из ранее разработанных версий кода (откатить изменения). Так, если бы вы в файловой системе постоянно делали резервное копирование (backup) ваших изменяемых данных, а в случае ошибок копировали из резерва. Git позволяет нам избежать ошибок при разработке и проектировании благодаря хранению всех версий.

GitHub — это крупнейший веб-сервис для хостинга IT-проектов и их совместной разработки. Репозитории открытых проектов в GitHub видны всем — можно делать копии основной ветки, чтобы что-то изменить и предложить свои изменения автору проекта. Еще используется для версионирование разработки приложений в команде.

Но у Git есть и другие преимущества, он позволяет:

  • Работать с удаленными репозиториями.
  • Сотрудничать с другими разработчиками, контролировать и сливать разные версии кода, работать одновременно.
  • Хранить историю изменений и видеть разницу нового кода и старого (difference).
  • Откатить (вернуть обратно) не всю версию программы, а её отдельных изменений (коммитов).
  • Управлять конфликтами при слиянии изменений.
  • Работать в ветках, что позволяет разрабатывать новые функциональности независимо от основной кодовой базы.
